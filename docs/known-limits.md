# Known Limits

- The hosted demo uses Netlify Functions rather than Python FastAPI because Netlify does not host persistent Python web servers.
- The graph is a production-style LangGraph concept surface rather than a deployed Python `langgraph` runtime.
- The hosted graph batches the ten pre-approval tool responses into one GLM-5-Turbo superstep so the SSE stream reaches the approval card before Netlify's synchronous function ceiling. Four additional tool endpoints remain available and are used post-approval or for direct demos.
- Tool responses are generated by GLM and vary by run, so exact fields can differ.
- Browser preview on plain Vite cannot call Netlify Functions; use Netlify Dev for live API testing.
- PDF export uses the browser print/save-to-PDF flow to avoid shipping a vulnerable client PDF generation dependency.
